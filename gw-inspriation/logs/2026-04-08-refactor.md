**设计要点：**
- Content 管线：Goldmark（MD→HTML）→ Bluemonday（`UGCPolicy` XSS 净化）→ 存库
- `ContentRaw` 字段保留原始 Markdown，供编辑时回显；渲染 HTML 存 `Content` 字段
- 热度公式（Hacker News 变体）：`HotScore(replies, votes int, createdAt) float64` = `(replies*2 + votes) / pow(ageHours+2, 1.5)` — 已导出，方便单元测试
- 楼层序号：事务内 `SELECT posts FOR UPDATE` + `Number = RepliesCount + 1`（悲观锁，并发安全）
- 帖子 Post 反规范化：`RepliesCount`、`LastReplyAt`、`LastReplyUser`（来自 Flarum 设计）
- `search_vector tsvector` 列带 `gorm:"-:migration"` 跳过 AutoMigrate；`MigrateSQL()` 返回 PostgreSQL 触发器 DDL，**需部署后手动执行一次**
- MVP 搜索降级：全文搜索不可用时退回 `ILIKE`
- `sensitive` 过滤（importcjj/sensitive）留作后续 TODO，MVP 未引入

**参考来源：** Flarum 的 RepliesCount/LastReplyAt 设计 + Gitea goldmark/bluemonday 管线

**验证：** `go build ./...` 通过。

**设计要点：**
- UNIQUE INDEX 通过原生 SQL 建立（GORM 不支持多列唯一索引 tag）
- `syncCount` 在 like 操作后原子更新目标表 `vote_up`（`GREATEST(0, vote_up ± 1)`）
- `syncCount` 静默失败，不影响主流程（comment/forum 表尚未创建时不报错）
- 匿名用户只读，写操作返回 401
- `favorite` 不影响 `vote_up`（收藏逻辑与热度排序分离）

**参考来源：** Gitea reaction.go 的 UNIQUE(s) 约束 + Artalk vote.go 的 syncCount 模式

## Task 6：Engine API 路由迁移 ✅

**完成时间：** 2026-04-10

**背景：** `engine/api/routes.go`（原 564 行）混合了玩家发现层（纯 DB 读）和执行层（需要 LLM/Pipeline）两类路由，违反职责单一原则，且导致 A-10（`comment_config`）无法在 engine 层合法 import `social/comment`。

**新建 `internal/platform/play/`：**

```
internal/platform/play/
├── handler.go   ← Handler + SessionCreator 接口 + publicGameView + 5 个 handler 方法
└── routes.go    ← RegisterPlayRoutes(rg, h)
```

**迁移的 5 条路由（从 engine/api/routes.go 移出）：**

| 路由 | 说明 |
|------|------|
| `GET /play/games` | 游戏列表，分页/标签/类型/排序 |
| `GET /play/games/:slug` | 游戏详情（slug 或 UUID），附加 comment_config |
| `GET /play/games/worldbook/:id` | 玩家只读世界书 |
| `GET /play/sessions` | 存档列表，直接 GORM 查询 |
| `POST /play/sessions` | 创建会话，调用 SessionCreator 接口 + 原子递增 play_count |

**关键设计决策：**
- `SessionCreator` 接口定义在 `platform/play/`，`GameEngine` 天然满足，无需修改 engine
- `publicGameView` 复制到 `platform/play/handler.go`，同时从 `engine_methods.go` 删除
- `GET /play/sessions` 改为直接 GORM 查询，不再依赖 `engine.ListSessions()`
- `platform/play/` 合法 import `social/comment`，A-10 在 `getGame()` 中实现

**文件变化：**
- `engine/api/routes.go`：564 → 437 行
- `engine/api/engine_methods.go`：删除 `publicGameView`（~28 行）
- `cmd/server/main.go`：新增 `playapi` import + `NewHandler` + `RegisterPlayRoutes`

**验证：** `go build ./...` + `go vet ./...` 均通过。
