# 创作者视角：从故事到游戏的全流程

> 2026-04-06
> 场景：我是一个只会写故事的作者，想把角色卡"食堂少女"和 Izumi 预设做成一个可游玩的游戏

---

## 我手里有什么

- 一张角色卡 PNG（食堂少女，黑发，暖光餐桌场景）
- 一套 Izumi 预设（65+ 条 Prompt，漫改叙事风格，对话驱动，分镜式写作）
- 脑海里的故事：玩家扮演常客，和食堂少女在日常中逐渐建立关系

我不懂技术，不知道 injection_order 是什么，不知道世界书怎么触发。

---

## CreationWorkshop 软件内的流程

### Step 1 — 打开 CreationWorkshop，创建新项目

我看到一个简洁的界面：
```
[新建项目]  [导入项目包]  [最近项目]
```

点"新建项目"，填写：
- 项目名：食堂物语
- 类型：text（纯文字）
- 一句话描述：在大学食堂和神秘少女的日常故事

软件自动调用 `POST /create/templates` 创建 GameTemplate。

### Step 2 — 选择预设

```
预设库
├── 推荐预设
│   ├── Izumi 漫改风格 ★★★★★
│   ├── 轻量叙事风格
│   └── 游戏机制风格
├── 导入预设
│   └── [从文件导入 ST 预设 JSON]
└── 从零开始（空白）
```

我选择 Izumi 漫改风格。软件调用 `POST /create/templates/:id/preset/import-st` 批量写入 PresetEntry。

导入后，软件用人话展示已导入的条目：
```
叙事规则（已导入 8 条）
├── 主提示 — 角色身份和协作关系     [启用 ✓]
├── 漫改风格 — 分镜式对话写作       [启用 ✓]
├── 扩写规则 — 最小化改写用户输入    [启用 ✓]
├── 短对话模式 — 纯 RP 简短回复     [禁用 ✗]
└── ...其他 4 条
```

我不需要知道 injection_order，软件用"叙事规则"这个分组帮我理解。

### Step 3 — 导入角色卡（可选）

```
角色设定
├── [导入角色卡 PNG]
├── [手动编写]
└── [让 AI 帮我写]
```

我拖入食堂少女的 PNG。软件调用 `POST /create/cards/import` 解析 tEXt chunk，提取 description/personality/scenario 写入 PresetEntry（injection_order 990-1009）。

导入后显示：
```
角色：食堂少女
├── 外貌描写 — 黑发，白衬衫，温暖的笑容    [编辑]
├── 性格 — 安静但话多起来停不下来           [编辑]
├── 场景 — 大学食堂，午后的阳光              [编辑]
└── 开场白 — "今天想吃什么？菜单上没有的..."  [编辑]
```

### Step 4 — 补充世界设定（可选）

```
世界书
├── [手动添加]
├── [让 AI 帮我写]  ← 用创作 Agent
└── [导入外部世界书]
```

我点"让 AI 帮我写"，进入创作 Agent 会话。AI 问我几个问题后，用 `worldbook_create` 工具帮我生成：
```
已创建词条：
├── "食堂" — 二层楼的老式大学食堂，窗边有一排老旧的木椅...
├── "少女" — 关于她的更多背景（触发关键词：名字、过去）
└── "常客" — 玩家角色的默认设定
```

### Step 5 — 试玩

软件内置试玩窗口。调用 `POST /play/sessions` 创建会话，直接开始对话。

```
[试玩窗口]
食堂少女："今天想吃什么？菜单上没有的我也能做哦。"

> 玩家输入：来一碗清汤面吧，多加点葱。

食堂少女：（低头在单子上划了一笔）"清汤面、多葱..."
（她抬起头，嘴角微微上扬）
"你每次都点这个。是真的喜欢，还是懒得看菜单？"
```

如果效果不满意，回到叙事规则调整。满意后进入下一步。

### Step 6 — 发布到 GameWorkshop

```
[发布]
├── 游戏名：食堂物语
├── 描述：在大学食堂和神秘少女的日常故事
├── 封面：（从角色卡 PNG 裁切）
├── 标签：日常 · 恋爱 · 纯文字
└── [确认发布]
```

软件调用：
1. `PATCH /create/templates/:id { "status": "published" }` 发布
2. `GET /create/templates/:id/export` 打包（可选备份）

底部三个按钮：
- 存档：`POST /play/sessions/:id/fork` 分叉当前会话
- 重试：`POST /play/sessions/:id/regen` 重新生成本回合
- 记录：查看对话历史 + Memory 摘要

### 存档与记录

```
我的游戏记录
├── 食堂物语
│   ├── 存档 1 — 回合 12 — "终于知道了她的名字"

│   └── 当前进度 — 回合 15
└── 异世界和平
    └── 当前进度 — 回合 6
```

每个存档是一个 GameSession，分叉创建新 Session 保留完整历史。

## 数据流全景

```
创作者（PC CreationWorkshop）
  │
  │ POST /create/templates         创建游戏
  │ POST /create/.../import-st     导入预设
  │ POST /create/cards/import      导入角色卡
  │ POST /create/.../lorebook      写世界书
  │ PATCH /create/templates/:id    发布
  │
  ▼
PostgreSQL（game_templates + preset_entries + worldbook_entries + ...）
  │
  │ GET /play/games                 获取已发布游戏列表
  │ POST /play/sessions             创建会话
  │ POST /play/sessions/:id/turn    游玩回合
  │ GET /play/sessions/:id/stream   SSE 流式输出
  │ POST /play/sessions/:id/fork    存档分叉
  │
  ▼
玩家（手机/Web GameWorkshop）
```

## MVP 范围

GameWorkshop 前端的 MVP 只需要：

1. **游戏列表页** — 展示 `status=published` 的 GameTemplate（封面、标题、类型、描述）
2. **游戏详情页** — 展示游戏信息 + "开始游玩"按钮
3. **游玩界面** — 对话式聊天 UI + 发送/重试/存档三个按钮
4. **我的记录** — Session 列表，可继续游玩或查看历史

不需要：
- 论坛/社交（MVM 分享层，稍后）
- 创作功能（CreationWorkshop 是独立软件）
- 复杂 UI 渲染（MVP 只做 text 类型）
